* Why AI for development?

- LLMs are just text generators basead on weight

- LLM is a developer tool, not a problem solver

- You (should) still need to think

- Works great for pair programming

- It's like a developer, it can be awesome but can mess things up too.

 [[file:images/funny-ai.png]]

* Why ECA?
- *Protocol*: Editors <-> LLMs     [[file:images/rationale.jpg]]
- *UX*: LLMs get better, UX not really

- *Multi-model*: with Local LLMs support

- *Fast:* Written in Clojure and compiled with GraalVM

- *Free:* Opensource

- *Transparent:* Prompt, contexts, _metrics_
